LECTURE 5 - CANONICAL COVERS

Canonincal Cover ≡ Minimal Cover
Schema Refinement = Can FD's F be minimized/more compact?

G is a canonical/mnimal cover of F when:

  1. G is equivalent to F (G ≡ F)
  2. G is minimal (if any part of G is removed G ≠ F)
  3. Every FD in G has a single attribute on the RHS.

* ( G ≡ F ) if ( F ⊨ G ) and ( G ⊨ F )
* multiple canonical covers can exists

A canonical cover G is minimal in two respects:

  1. Every FD in G is “required” in order for G to be equivalent to F.
  2. Every FD in G is as “small” as possible (only one attribute RHS)

Computing Canonical Cover

  1. Simple form for every FD's (i.e., one attribute on the RHS)
  2. Check for Retundant Attributes: (i.e. Minimize/check for LEFT Redundancy)
    • A FD in the set is redundant if it can be derived from the other FD's in the set.
    • For every FD X → A in G check if closure of a subset of X determines A. If so, remove redundant attribute(s) from X's:
    • ex: AB → C, check if A or B is redundant ?
      • Check A+
      • Check B+
      • say A+ = AC
      • C ∈ A+ then B+ is Redundant
      • i.e. C can be drived without the need for B
  3. Check for Redundant FD's: (i.e. Delete RIGHT Redundancy)
    • ( G - {X → A} )+ ≡ F+ ?
    • For every FD X → A in G
      • Remove {X → A} from G; call the result G’
      • Compute X+ under G’
      • If A ∈ X+ under G’, then X → A is redundant and hence remove X → A from G.

* ORDER MATTERS

Example:

This example explains computing canonical covers well, just dont move to the next line until you know exactly whats going on first.
R = { A, B, C, D, E, H }
F = { A → B , DE → A , BC → E , AC → E , BCD → A , AED → B }

  1. Put FDs in simple form:

    • G = { A → B, DE → A, BC → E, AC → E, BCD → A, AED → B }
    • Already in simple form (RHS = singletons)

  2. Check for Retundant Attributes:

    • G = { A → B, DE → A, BC → E, AC → E, BCD → A, AED → B }
    • A → B
      • obviously OK (no left redundancy)
    • DE → A
      • D+ = D
      • E+ = E
      • OK (no left redundancy)
    • BC → E
      • B+ = B
      • C+ = C
      • OK (no left redundancy)
    • AC → E
      • A+ = AB
      • C+ = C
      • OK (no left redundancy)
    • BCD → A
      • B+ = B
      • C+ = C
      • D+ = D
      • BC+ = BCE
      • CD+ = CD
      • BD+ = BD
      • OK (no left redundancy)
    • AED → B
      • A+ = AB
      • E = redundant
      • D = redundant
      • B can be inferred without the need for E & D
      • E & D can be removed from AED → B
      • G = { A → B, DE → A, BC → E, AC → E, BCD → A, A → B}
      • G = { DE → A, BC → E, AC → E, BCD → A, A → B}

  3. Check for Redundant FD's:

    • G = { DE → A, BC → E, AC → E, BCD → A, A → B }
    • DE → A
      • Remove DE → A from G
      • G’ = { BC → E, AC → E, BCD → A, A → B }
      • Compute DE+ under G’
      • DE+ = DE
      • Since A ∉ DE
      • DE → A is not redundant
      • G = { DE → A, BC → E, AC → E, BCD → A, A → B }
    • BC → E
      • Remove BC → E from G
      • G’ = { DE → A, AC → E, BCD → A, A → B }
      • Compute BC+ under G’
      • BC+ = BC
      • Since E ∉ BC
      • BC → E is not redundant
      • G = { DE → A, BC → E, AC → E, BCD → A, A → B }
    • AC → E
      • Remove AC → E from G
      • G' = { DE → A, BC → E, BCD → A, A → B }
      • Compute AC+ under G’
      • AC+ = ACBE
      • Since E ∈ ACBE
      • AC → E IS REDUNDANT, remove it from G
      • G = { DE → A, BC → E, BCD → A, A → B }
    • BCD → A
      • Remove BCD → A from G
      • G' = { DE → A, BC → E, A → B }
      • Compute BCD+ under G’
      • BCD+ = BCDEA
      • Since A ∈ BCDEA
      • BCD → A IS REDUNDANT, remove it from G
      • G = { DE → A, BC → E, A → B }
    • A → B
      • Remove A → B from G
      • G' = { DE → A, BC → E}
      • Compute A+ under G’
      • A+ = A
      • Since B ∉ A
      • A → B is not redundant
      • G = { DE → A, BC → E, A → B }

Thus a Minimal Cover of F is G = { DE → A, BC → E, A → B }
